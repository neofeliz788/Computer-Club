Конфликт одновременного бронирования

Дефект заведен в GitHub Issues
Issue #1

Описание

При одновременной попытке бронирования одного компьютера двумя клиентами система позволяет создать две активные брони. Это критическая ошибка, которая может привести к конфликтам между пользователями и некорректной работе системы бронирования.

Шаги воспроизведения

  1. Открыть два разных браузера (например, Chrome и Firefox) или две приватные вкладки.
  2. В обоих браузерах открыть систему бронирования компьютеров.
  3. В каждом браузере выбрать один и тот же компьютер на одно и то же время.
  4. Нажать кнопку "Забронировать" практически одновременно в обоих браузерах.
  5. Проверить результат бронирования в административной панели или базе данных.

Ожидаемый результат

Ожидаемый результат

Только одна бронь должна быть создана. Система должна предотвращать конфликты бронирования через механизм блокировок или проверок в реальном времени.

Фактический результат

Фактический результат

Создаются две активные брони на один компьютер в одно и то же время. Это указывает на отсутствие корректной обработки конкурентных запросов.

Метки (Labels)

bug priority-high database

Рекомендации по исправлению

  • Реализовать механизм оптимистичных или пессимистичных блокировок на уровне базы данных.
  • Добавить проверку наличия активной брони перед созданием новой в рамках транзакции.
  • Внедрить очередь запросов на бронирование для обработки конкурентных попыток.
  • Добавить индикацию пользователю, что компьютер уже забронирован, если попытка бронирования не удалась.
