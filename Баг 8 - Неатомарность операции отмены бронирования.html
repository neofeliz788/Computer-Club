Неатомарность операции отмены бронированияCritical

Дефект заведен в GitHub Issues
Issue #8

Описание

При отмене бронирования операция обновления статуса компьютера и создание записи об отмене выполняются не в одной транзакции. Это может привести к несогласованному состоянию данных в случае сбоев.

Шаги воспроизведения

  1. Создать бронирование компьютера на определенное время.
  2. Инициировать операцию отмены бронирования.
  3. Во время выполнения операции вызвать искусственный сбой сети или базы данных сразу после обновления статуса компьютера, но до создания записи об отмене.
  4. Проверить состояние системы после восстановления работы.
  5. Повторить сценарий с сбоем на разных этапах операции.

Диаграмма выполнения операции отмены

Начальное состояние
Компьютер: занят
Отмена: не создана
→
1. Обновление статуса компьютера
UPDATE Computers SET Status = 'available'
→

Сбой сети / БД
→
Конечное состояние
Компьютер: свободен
Отмена: не создана

Бизнес-последствия

Расхождение в финансовой отчетности. Система считает компьютер свободным (можно забронировать снова), но не имеет записи об отмене первоначальной брони, что приводит к:

  • Некорректному учету доходов (отмена не учитывается)
  • Возможности двойного бронирования одного компьютера
  • Ошибкам в аналитических отчетах
  • Проблемам с аудитом и соответствием требованиям
  • Финансовым потерям при неправильном учете

Ожидаемый результат

Ожидаемый результат

Либо все изменения (обновление статуса компьютера и создание записи об отмене) успешно применяются, либо все изменения откатываются. Система должна гарантировать атомарность операции.

Фактический результат

Фактический результат

Компьютер помечен как свободный (статус обновлен), но запись об отмене бронирования не создана. Система находится в несогласованном состоянии.

Решение 1: Использование транзакции БД

public void CancelBooking(int bookingId) {
    using (var connection = new SqlConnection(connectionString)) {
        connection.Open();
        
        // Начало транзакции
        using (var transaction = connection.BeginTransaction()) {
            try {
                // 1. Обновление статуса компьютера
                var updateComputerCmd = new SqlCommand(
                    "UPDATE Computers SET Status = 'available' WHERE Id = @computerId", 
                    connection, transaction);
                updateComputerCmd.Parameters.AddWithValue("@computerId", GetComputerId(bookingId));
                updateComputerCmd.ExecuteNonQuery();
                
                // 2. Создание записи об отмене
                var insertCancelCmd = new SqlCommand(
                    "INSERT INTO Cancellations (BookingId, CancelDate, Reason) VALUES (@bookingId, @date, @reason)", 
                    connection, transaction);
                insertCancelCmd.Parameters.AddWithValue("@bookingId", bookingId);
                insertCancelCmd.Parameters.AddWithValue("@date", DateTime.Now);
                insertCancelCmd.Parameters.AddWithValue("@reason", "User cancelled");
                insertCancelCmd.ExecuteNonQuery();
                
                // Фиксация транзакции
                transaction.Commit();
            }
            catch (Exception ex) {
                // Откат транзакции при ошибке
                transaction.Rollback();
                throw;
            }
        }
    }
}

Решение 2: Транзакция через Entity Framework

public void CancelBooking(int bookingId) {
    using (var context = new BookingDbContext()) {
        // Начало транзакции
        using (var transaction = context.Database.BeginTransaction()) {
            try {
                // 1. Обновление статуса компьютера
                var computerId = context.Bookings
                    .Where(b => b.Id == bookingId)
                    .Select(b => b.ComputerId)
                    .FirstOrDefault();
                    
                var computer = context.Computers.Find(computerId);
                if (computer != null) {
                    computer.Status = "available";
                }
                
                // 2. Создание записи об отмене
                var cancellation = new Cancellation {
                    BookingId = bookingId,
                    CancelDate = DateTime.Now,
                    Reason = "User cancelled"
                };
                context.Cancellations.Add(cancellation);
                
                // Сохранение изменений и фиксация транзакции
                context.SaveChanges();
                transaction.Commit();
            }
            catch (Exception ex) {
                transaction.Rollback();
                throw;
            }
        }
    }
}

Решение 3: Паттерн Saga для распределенных транзакций

// Используется в микросервисной архитектуре
public async Task CancelBookingAsync(int bookingId) {
    var saga = new CancelBookingSaga(bookingId);
    
    try {
        // Шаг 1: Резервирование отмены
        await saga.ReserveCancellationAsync();
        
        // Шаг 2: Освобождение компьютера
        await saga.FreeComputerAsync();
        
        // Шаг 3: Фиксация отмены
        await saga.ConfirmCancellationAsync();
        
        // Все шаги успешны
        await saga.CompleteAsync();
    }
    catch (Exception ex) {
        // Компенсирующие транзакции
        await saga.CompensateAsync();
        throw;
    }
}

// Каждый шаг Saga имеет компенсирующую операцию
public class CancelBookingSaga {
    public async Task CompensateAsync() {
        // Отмена всех выполненных шагов в обратном порядке
        if (_cancellationReserved) await CompensateCancellationAsync();
        if (_computerFreed) await CompensateComputerFreeAsync();
    }
}

Решение 4: Использование Outbox Pattern

public async Task CancelBookingAsync(int bookingId) {
    using (var context = new BookingDbContext()) {
        using (var transaction = await context.Database.BeginTransactionAsync()) {
            try {
                // 1. Обновление статуса компьютера
                var computer = await GetComputerForBookingAsync(bookingId);
                computer.Status = "available";
                
                // 2. Создание записи об отмене
                var cancellation = new Cancellation { /* ... */ };
                context.Cancellations.Add(cancellation);
                
                // 3. Запись события в Outbox в той же транзакции
                var outboxMessage = new OutboxMessage {
                    Type = "BookingCancelled",
                    Data = JsonSerializer.Serialize(new {
                        BookingId = bookingId,
                        CancelledAt = DateTime.UtcNow
                    }),
                    CreatedAt = DateTime.UtcNow
                };
                context.OutboxMessages.Add(outboxMessage);
                
                // Все операции фиксируются вместе
                await context.SaveChangesAsync();
                await transaction.CommitAsync();
            }
            catch (Exception ex) {
                await transaction.RollbackAsync();
                throw;
            }
        }
    }
}
