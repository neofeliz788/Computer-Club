Проблемы с кодировкой при экспорте отчетов в CSV

Дефект заведен в GitHub Issues
Issue #6

Описание

При экспорте отчетов, содержащих кириллические символы, в формате CSV возникают иероглифы вместо русских букв. Проблема проявляется при открытии CSV-файлов в Microsoft Excel и других программах, которые по умолчанию используют кодировку Windows-1251 или другую, отличную от UTF-8.

Шаги воспроизведения

  1. Создать бронирование с клиентом "Иванов Иван".
  2. Перейти в раздел отчетов административной панели.
  3. Выбрать отчет за текущий день.
  4. Нажать кнопку "Экспорт в CSV".
  5. Сохранить файл на компьютер.
  6. Открыть CSV-файл в Microsoft Excel.
  7. Проверить отображение кириллических символов.

Техническое объяснение проблемы

Проблема возникает из-за несоответствия кодировок:

  • Сайт использует UTF-8 для хранения и отображения данных
  • CSV-файл генерируется в UTF-8 без BOM (Byte Order Mark)
  • Microsoft Excel по умолчанию пытается открыть CSV в Windows-1251 (на русскоязычных Windows)
  • Excel неправильно интерпретирует UTF-8 байты как Windows-1251, что приводит к "кракозябрам"

Возможные решения

Решение Описание Плюсы Минусы
UTF-8 с BOM Добавить BOM (EF BB BF) в начало CSV-файла Простая реализация, Excel правильно определяет кодировку Не все программы поддерживают BOM
Windows-1251 Генерировать CSV в кодировке Windows-1251 Идеально для русскоязычного Excel Не поддерживает другие языки
Указание кодировки в имени Использовать filename_utf8.csv или filename_win1251.csv Пользователь понимает, какую кодировку использовать Требует действий от пользователя
Excel-совместимый CSV Использовать разделитель ";" и кодировку Windows-1251 Наиболее совместимо с Excel Менее стандартный формат CSV

Реализация решения

Решение 1: UTF-8 с BOM (рекомендуется)

// Java пример
response.setContentType("text/csv; charset=UTF-8");
response.setHeader("Content-Disposition", "attachment; filename=report.csv");

// Добавляем BOM
OutputStream out = response.getOutputStream();
out.write(0xEF);
out.write(0xBB);
out.write(0xBF);

// Пишем данные
PrintWriter writer = new PrintWriter(new OutputStreamWriter(out, "UTF-8"));
writer.println("Client,Service,Date");
writer.println("Иванов Иван,Аренда компьютера,2024-01-15");
writer.close();

Решение 2: Windows-1251 для русскоязычных пользователей

// Java пример
response.setContentType("text/csv; charset=Windows-1251");
response.setHeader("Content-Disposition", "attachment; filename=report.csv");

// Конвертируем в Windows-1251
String csvContent = "Client,Service,Date\nИванов Иван,Аренда компьютера,2024-01-15";
byte[] windows1251Bytes = csvContent.getBytes("Windows-1251");

// Отправляем
OutputStream out = response.getOutputStream();
out.write(windows1251Bytes);
out.close();

Решение 3: Детекция и выбор кодировки

// Определяем предпочтительную кодировку на основе заголовков или настроек
String userAgent = request.getHeader("User-Agent");
String preferredEncoding = "UTF-8";

if (userAgent != null && userAgent.contains("Windows")) {
    // Для Windows пользователей используем Windows-1251
    preferredEncoding = "Windows-1251";
}

response.setContentType("text/csv; charset=" + preferredEncoding);
response.setHeader("Content-Disposition", 
    "attachment; filename=report_" + preferredEncoding.toLowerCase() + ".csv");

// Генерация CSV с выбранной кодировкой
